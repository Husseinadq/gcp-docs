## TL;DR

Tutorials are for learning, not for covering every option.

If a page is a tutorial, it should:

- have one outcome
- assume as little as possible
- explain the reason for each step
- leave the reader with something working

## What you should expect from this section

Every tutorial in this section should give you:

- a clear end state
- a short stack decision
- a small number of services
- checkpoints that confirm you are still on track

## Start here

Read [Ship Your First Web App on GCP](/docs/learn/ship-your-first-web-app-on-gcp) first. It is the shortest path from confusion to a working platform model.
